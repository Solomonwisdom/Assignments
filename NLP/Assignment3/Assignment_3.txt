# 作业3综述
Assignment 3任务为自然语言通顺与否的判定，即给定一个句子，要求判定所给的句子是否通顺。作业3仍然采用在线competition形式，除了提供的训练集之外，可以使用外部数据（如无标注的文本语料）。Competition结束后需要在两天内将代码和说明文档提交至ftp服务器，作业3成绩会根据作业报告质量和leaderboard排名综合给出，本次作业强调对问题的深入分析、问题的解决方案以及对结果的详细分析，在使用machine learning的同时也可以考虑特征工程，这些内容需要在报告中都有所体现。作业报告需要提供源码和文档，文档可分为问题深入分析，技术方案指定，遇到的问题及解决方案，用到的数据，性能评价，结果分析，将来可能的改进。需要给出程序的运行环境，执行方法及参数。

# 数据格式
###训集集
每个训练样例包含id、句子、标签三个元素，分别对应句子的id、句子内容以及句子是否通顺，用'\t'分隔，标签为0表示通顺，为1时表示不通顺，如下面的样例：
```txt
1	你喜欢听什么	0
2	的生日怎么样	1
3	两个还有一个两个三个	1
```

###测试集
每个测试样例包含id、句子两个元素，分别对应句子的id、句子内容，用'\t'分隔，如下面的样例：
```txt
1	鸦片战争后,厦门成为1842年"南京条约"后首批对外开放的通商口岸之一。
2	脑供血严重不足
3	听写练习,我
```

# 评分计算方式
F1-score，具体计算逻辑可以参见脚本metrics.py。

# 提交格式
每行需要包含id和标签两个元素，用'\t'分隔，id为测试集中句子的id，标签为对应句子的预测结果，为0表示通顺，为1时表示不通顺。提交的预测结果数目需要和测试集数目相同，数目不同时提交无效。
提交样例文件参见sample.test.prediction.txt。
